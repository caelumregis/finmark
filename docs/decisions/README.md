# Architecture decision records

Use ADRs for decisions that are difficult to reverse or affect multiple parts of Finmark.

## Index

| ADR | Decision | Status |
| --- | --- | --- |
| [0000](0000-template.md) | ADR template | Template |
| [0001](0001-modular-monolith.md) | Use a modular monolith with event-driven workers | Proposed |

## Workflow

1. Copy `0000-template.md` to the next numbered file.
2. Describe the context and viable options.
3. Record the decision, consequences, owner, and date.
4. Set the status to Proposed, Accepted, Superseded, or Rejected.
5. Add the ADR to this index.

Do not rewrite an accepted ADR to change history. Create a new ADR and mark the earlier record superseded.

# Implementation guides

These guides document FinMark’s implementation branch by branch and commit by commit. They explain what was built, why each decision was made, how important code works, and how the result was verified.

## Guides

| Guide | Branch | Purpose |
| --- | --- | --- |
| [Repository foundation](01-repository-foundation.md) | `chore/repository-foundation` | Establish the npm workspace and repository conventions |

## Documentation standard

Each implementation guide should include:

1. The branch goal
2. Commits included
3. Files introduced or changed
4. Important code snippets
5. Explanation of the implementation
6. Commands used
7. Verification results
8. Challenges encountered
9. Possible refinements
10. Lessons learned
